# IHttpContext::GetResponseHeadersSent Method
Determines whether the HTTP response headers have been sent to the client.  
  
## Syntax  
  
```cpp  
virtual BOOL GetResponseHeadersSent(  
   VOID  
) const = 0;  
```  
  
#### Parameters  
 This method takes no parameters.  
  
## Return Value  
 `true` if the HTTP response headers have been sent; otherwise, `false`.  
  
## Remarks  
 The `GetResponseHeadersSent` method determines whether another module has already sent the HTTP response headers to the client (for example, another HTTP module has called the [IHttpResponse::Flush](../../../webdevelopment-reference\native-code-api\webdev-native-api-reference/ihttpresponse-flush-method.md) method). The `GetResponseHeadersSent` method is especially useful when you are creating an HTTP module that writes values to the headers, because it enables you to have deterministic control over the information that your HTTP module outputs to a Web client.
